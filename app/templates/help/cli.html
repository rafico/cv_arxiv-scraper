CLI Commands

The web UI is enough for most users. Use the CLI if you want automation, historical backfills, headless operation, or shell scripting.

After installing with pip install -e ., these commands are available globally:

Command Description
cv-arxiv-scrape Run one scrape from the terminal and print matched papers
cv-arxiv-digest Build or send the email digest. Useful flags: --dry-run, --send-only
cv-arxiv-sync Historical sync for backfilling older papers. Flags: --from, --to, --category, --chunk-days
cv-arxiv-backfill Run enrichment backfills on papers already in your database

Standalone scripts (python scrape_cli.py, python export_cli.py, etc.) also work without installing the package.

Enrichment Backfills

Use these when papers were added before a feature was enabled, or when you want to rebuild missing metadata later.

Subcommand What it does
embeddingsBackfill missing SPECTER2 embeddings
index-rebuildRebuild the full FAISS index from the database
citationsFetch citation counts from Semantic Scholar
openalexFetch OpenAlex metadata (topics, OA status)
thumbnailsGenerate PDF first-page thumbnails
allRun all of the above

All subcommands accept --batch-size N and --delay SECONDS.

Scheduled Scrapes

The app can scrape automatically at a set time each day, without needing cron.

Add a scheduler section to your config.yaml:

scheduler:
  enabled: true
  scrape_time: "08:00"   # 24-hour format, local time

Alternatively, use cron:

0 8 * * * cd /path/to/cv_arxiv-scraper && ~/venv/bin/python -m app.cli.scrape

Multiple Feed Sources

Monitor additional arXiv categories or custom RSS feeds beyond cs.CV. Add, remove, and toggle feed sources from Settings or via the /api/feed-sources API.

AI Summaries (LLM)

Optional feature

The app works fine without AI. Summaries and topic tags are generated extractively by default. Enabling LLM adds richer summaries and an AI relevance score.

When enabled, the LLM reads each paper’s title and abstract to produce:

  • A plain-language TL;DR summary describing what the paper does and why it matters (shown on each paper card in the dashboard)
  • A relevance score (0–10) relative to your research interests

Without LLM, the app still generates an extractive summary from the abstract. You can control how many lines of the summary are visible on paper cards via Settings → Ranking → Summary lines.

Setup

1

Enable in Settings

Go to Settings → LLM / AI and toggle LLM on.

2

Choose a provider

Supported providers: OpenRouter for a hosted model, or Ollama for a local model. Add the API key or local base URL.

3

Run a scrape

AI summaries and relevance scores are generated during the scrape. Papers already in the database are not automatically re-processed unless you run a backfill workflow.

config.yaml example

llm:
  enabled: true
  provider: openrouter         # or "ollama"
  model: google/gemma-4-31b-it:free
  # base_url: http://localhost:11434/v1   # for Ollama
  max_concurrent: 4
